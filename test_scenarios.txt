## 3. СЦЕНАРИИ С ОШИБКАМИ (проверка обработки ошибок)

### 3.1. Ошибки конфигурации

# Несуществующий пакет
{
  "package_name": "nonexistent-package-12345xyz",
  "repository_url": "https://registry.npmjs.org",
  "repository_mode": "remote"
}

# Неправильный URL репозитория
{
  "package_name": "react",
  "repository_url": "invalid-url-without-protocol",
  "repository_mode": "remote"
}

# Неправильный режим работы
{
  "package_name": "react",
  "repository_url": "https://registry.npmjs.org",
  "repository_mode": "invalid_mode"
}

# Пустое имя пакета
{
  "package_name": "",
  "repository_url": "https://registry.npmjs.org",
  "repository_mode": "remote"
}

# Пустой URL репозитория
{
  "package_name": "react",
  "repository_url": "",
  "repository_mode": "remote"
}

# Пустой режим
{
  "package_name": "react",
  "repository_url": "https://registry.npmjs.org",
  "repository_mode": ""
}

### 3.2. Ошибки тестового режима

# Несуществующий тестовый репозиторий
{
  "package_name": "react",

## 5. КОМАНДЫ ДЛЯ БЫСТРОГО ТЕСТИРОВАНИЯ

# Для быстрого переключения между конфигурациями создайте отдельные файлы:

# config_react.json
echo '{"package_name": "react", "repository_url": "https://registry.npmjs.org", "repository_mode": "remote"}' > config_react.json

# config_express.json
echo '{"package_name": "express", "repository_url": "https://registry.npmjs.org", "repository_mode": "remote"}' > config_express.json

# config_webapp_test.json
echo '{"package_name": "webapp", "repository_url": "test_repository", "repository_mode": "test"}' > config_webapp_test.json

# Затем копируйте нужный конфиг:
cp config_react.json config.json && python dependency_visualizer.py
cp config_express.json config.json && python dependency_visualizer.py
cp config_webapp_test.json config.json && python dependency_visualizer.py

## 6. ОЖИДАЕМЫЕ РЕЗУЛЬТАТЫ

✅ РАБОЧИЕ СЦЕНАРИИ:
- React: зависимости в версиях до 19.x
- Express: 20+ зависимостей
- Webpack: 100+ зависимостей
- Webapp (тест): 3 зависимости (react, express, axios)

✅ ПАКЕТЫ БЕЗ ЗАВИСИМОСТЕЙ:
- Lodash, Chalk и др.: "Прямые зависимости не найдены"

❌ СЦЕНАРИИ С ОШИБКАМИ:
- Корректные сообщения об ошибках
- Не аварийное завершение

## 7. ПОСЛЕДОВАТЕЛЬНОСТЬ ТЕСТИРОВАНИЯ

1. Сначала тестовый режим (webapp)
2. Затем простые пакеты (express)
3. Сложные пакеты (webpack)
4. Пакеты без зависимостей (lodash)
5. Сценарии с ошибками

## 8. ПРИМЕЧАНИЯ

- Для тестового режима должен существовать каталог test_repository/
- Некоторые пакеты (как React) могут не иметь зависимостей в последних версиях
- Инструмент автоматически ищет версии с зависимостями
- Все ошибки должны обрабатываться корректно без падения приложения
